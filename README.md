## 🧪 what is this

A sandbox for testing **animated widgets that work inside GitHub READMEs**. GitHub strips `<script>`, `<style>`, and iframes from markdown, so every "animation" here is really one of three tricks:

1. **Animated SVGs** served by an external service (SMIL/CSS animations survive inside `<img>`)
2. **GIFs** — the classic
3. **GitHub Actions** that periodically re-render an SVG and commit it to the repo

Each section below notes the service that powers it.

## 🐍 contribution snake

Generated by [Platane/snk](https://github.com/Platane/snk) via the workflow in [`.github/workflows/snake.yml`](.github/workflows/snake.yml). It runs daily, renders the snake eating my contribution graph, and pushes the SVGs to the `output` branch. Light/dark variants swap automatically via the `<picture>` tag.

## 📝 widget cheat sheet

| Widget | Service | Animated how |
|---|---|---|
| Wave header | [capsule-render](https://github.com/kyechan99/capsule-render) | SVG animation, rendered on request |
| Typing text | [readme-typing-svg](https://github.com/DenverCoder1/readme-typing-svg) | SVG `<animate>` loop |
| Snake | [Platane/snk](https://github.com/Platane/snk) | GitHub Action re-renders daily |
| Stats / streak | [github-readme-stats](https://github.com/anuraghazra/github-readme-stats) | Re-rendered per page load |
| Activity graph | [activity-graph](https://github.com/Ashutosh00710/github-readme-activity-graph) | Re-rendered per page load |
